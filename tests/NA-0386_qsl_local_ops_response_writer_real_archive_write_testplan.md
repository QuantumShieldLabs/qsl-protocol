# NA-0386 QSL Local Ops Response Writer Real-Archive Write Testplan

Goals: G1, G2, G3, G4, G5

## Objective

Validate that NA-0386 implements a bounded, dual-gated, no-secret real response
archive smoke write harness while preserving NA-0384 temp-output behavior and
without runtime, workflow, dependency, backup-config, or public-claim drift.

## Protected invariants

- READY_COUNT remains `1`.
- READY remains `NA-0386` until optional closeout.
- NA-0385 remains DONE.
- D-0752 exists once.
- D-0753 exists once.
- D-0754 exists once after implementation.
- Exactly one helper-created real archive smoke file is written.
- Existing archive files are not overwritten, deleted, truncated, renamed, or
  edited.
- No response, directive, journal, or local-history index is created.

- `scripts/ci/qsl_codex_response_writer.py`

## Forbidden scope

Forbidden paths include `.github/**`, workflows, Cargo/dependency files,
runtime/service/protocol/crypto/auth/state-machine files, qshield runtime,
qsl-server, qsl-attachments, qsc-desktop, website, docs/public, README,
START_HERE, backup scripts/timers/fstab/source lists/system services,
`scripts/ci/qsl_evidence_helper.py`, `scripts/ci/qsl_bounded_check_poll.py`,
`scripts/ci/qsl_directive_manifest_validate.py`,
`scripts/ci/public_safety_gate.py`, `/srv/qbuild/tools/**`, response indexes,
directive/journal indexes, and `/home/victor/work/qsl/codex/**` except the
single helper-created smoke file and final D205 response file.

## Helper CLI requirements

- `write`, `dry-run`, and `validate` support `--allow-real-archive-output`.
- Real archive output fails closed without the CLI gate.
- Temp-output mode remains available without the CLI gate.
- JSON summary output remains valid.

## Real archive gate requirements

- Metadata must set `output_mode: real_archive_smoke`.
- Metadata must set `allow_real_archive_output: true`.
- Metadata must set `no_secret_required: true`.
- Output directory must resolve exactly to `/home/victor/work/qsl/codex/responses`.
- Real smoke metadata target must be NA-0386 and directive suffix D205.

## Fixture requirements

Fixture cases must be deterministic, local-only, and no-network. The matrix must
cover temp-output compatibility, real-gate validation, exact-path dry-run,
collision simulation, missing gates, wrong path, no-secret rejection, overwrite
rejection, index/delete request rejection, malformed metadata, invalid target,
invalid directive suffix, invalid timestamp, and missing required section.

## Positive validation requirements

- Existing NA-0384 fixture matrix passes.
- NA-0386 fixture matrix passes.
- Real archive metadata/body validates with both gates present.
- Real archive dry-run computes the expected path without writing.
- Collision suffix logic still produces `_r2` in simulation.

## Negative / fail-closed requirements

The helper must reject:

- missing CLI gate;
- missing metadata gate;
- wrong output directory;
- `no_secret_required: false`;
- high-confidence secret pattern;
- existing output with collision disabled;
- index request;
- delete/cleanup request;
- malformed metadata;
- invalid target;
- invalid directive suffix;
- invalid timestamp;
- missing required section.

## Real archive smoke requirements

The smoke file must be generated by the helper, under the real response archive,
and must include:

- `NA-0386 SYNTHETIC REAL-ARCHIVE SMOKE FILE — NOT A FINAL CODEX RESPONSE`;
- `This smoke artifact proves archive-write mechanics only; it does not prove disaster recovery.`;
- `No production, public-internet, anonymity, metadata-free, untraceable, external-review, off-host-backup-complete, or disaster-recovery-complete claim is made.`

## No-overwrite / no-delete / no-index requirements

- Before/after archive manifests must show exactly one new file.
- Existing archive checksums must not change.
- A no-collision check against the written smoke file must reject.
- A collision-enabled dry-run may compute `_r2` but must not write it.
- No index or catalog path may appear.

## No-network / no-mutation requirements

The helper must remain Python standard library only and must not call GitHub,
network, shell, git, or subprocess APIs. No existing archive file may be mutated.

## No-secret requirements

The smoke body and metadata must pass the helper no-secret scan before write.
Secret sentinel fixture must fail before write.

## Backup-impact requirements

Evidence must record the response archive source-list coverage, mounted
same-host backup state, latest daily snapshot, and the caveat that the new smoke
file is not proven restored or snapshotted unless a later authorized backup
proves it. Same-host continuity must not be described as disaster recovery.

## Public-claim boundary requirements

Evidence and smoke content must not introduce production-readiness,
public-internet-readiness, external-review-complete, anonymity, metadata-free,
untraceable, off-host-backup-complete, or disaster-recovery-complete claims.

## Successor selection requirements

If the harness succeeds, select exactly:

`NA-0387 -- QSL Local Ops Response Archive Index and History Catalog Authorization Plan`

Do not implement NA-0387.
